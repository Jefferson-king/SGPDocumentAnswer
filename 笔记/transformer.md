Transformer 本质上是一种“让每个词都能直接看全局上下文，再决定自己该怎么表示”的模型架构。它最重要的突破，是不用像 RNN 那样按顺序一个词一个词处理，而是靠 self-attention 一次性看整段内容，所以更适合并行训练，也更擅长建模长距离依赖。

它的核心是 attention。你可以把一句话里的每个 token 都想成一个学生，大家在开会。每个 token 都会生成三份信息：

Q，我现在想找什么信息
K，我身上有什么信息可供别人匹配
V，如果别人关注我，我真正提供出去的内容
然后模型会用 Q 去和所有 token 的 K 做相似度计算，决定“我现在应该重点看谁”，再把这些 token 的 V 按权重加权求和，得到当前 token 的新表示。这个过程就是 attention。公式上最经典的就是：

Attention(Q, K, V) = softmax(QK^T / sqrt(d_k))V

这里除以 sqrt(d_k) 是为了防止数值太大，softmax 之后太极端。

multi-head attention 可以理解成“让模型从多个角度同时看关系”。一个头可能更关注语法主谓关系，一个头更关注指代，一个头更关注远距离依赖。最后把多个头的信息拼起来，表示能力会更强。

但如果只有 attention，还不知道词序。因为 “我爱你” 和 “你爱我” 词是一样的，顺序不同，意思完全不同。所以 Transformer 要加 position encoding，也就是位置编码，把“第几个词”这件事编码进去，让模型知道顺序信息。

从整体结构看，经典 Transformer 有 Encoder 和 Decoder。早期机器翻译常用完整的 encoder-decoder。现在很多大语言模型，比如 GPT，主要是 decoder-only 结构，本质上做的是“根据前文预测下一个 token”。

为什么 Transformer 这么强？因为它有三个优势：

能并行训练，扩展性很强
能直接看全局上下文，长距离建模能力好
随着数据、参数、算力增加，效果能持续提升
它也有明显缺点：

attention 计算复杂度通常和上下文长度平方相关，长文本成本高
它并不天然“理解世界”，只是学到了很强的统计模式
上下文再长也不是无限的，超出窗口仍然会丢信息
所以我对 Transformer 的一句话理解是：

它不是在“按顺序记住句子”，而是在“让每个 token 动态决定该关注上下文里的谁”，再通过层层堆叠，把原始文本变成高质量的上下文表示，最终完成预测。
