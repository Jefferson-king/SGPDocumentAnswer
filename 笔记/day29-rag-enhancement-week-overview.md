# 增强版 RAG 一周补强计划｜2026-04-13 ~ 2026-04-19

## 本周目标

把当前项目从“基础向量检索问答”升级为“可写进简历、可演示、可解释”的增强版 RAG 系统。

最终形成一版更完整的主链路：

文档上传 -> 结构化解析 -> chunk/父块入库 -> 混合召回 -> rerank -> final context -> 问答控制层 -> 结构化输出 -> 前端展示

## 当前项目基础

- 当前技术栈：`Node.js + Express + Vue`
- 当前已具备：文档上传、PDF/TXT 解析、chunk 切分、向量检索、基础问答、引用返回、流式输出、多轮问答
- 当前缺口：BM25 混合召回、rerank、父块/页级召回、结构化解析、richer metadata、问答控制层、结构化输出约束

## 本周验收标准

- 检索链路已升级为 `vector + BM25 + rerank`
- 支持 `chunk + 父块/页级召回`
- 文档解析结果能区分标题、段落、表格、页码
- chunk 元数据更完整
- 问答支持多类问题模板
- 输出统一为 `answer / citations / summary / confidence`
- 能录一版完整演示视频，并能支撑简历里的项目描述

## 本周每天安排

1. `Day 30`：梳理现有链路、拆模块边界、准备最小评测集
2. `Day 31`：接入 BM25，完成混合召回
3. `Day 32`：补 `topK recall -> rerank -> final context`
4. `Day 33`：补 chunk + 父块/页级召回 与 richer metadata
5. `Day 34`：把纯文本解析升级为结构化 block 解析
6. `Day 35`：增加问答控制层与问题模板
7. `Day 36`：统一结构化输出、加校验、做演示收尾

## 推荐执行原则

- 先让每个能力“可跑通”
- 再让它“可解释”
- 再让它“可演示”
- 最后再考虑更重的框架和更复杂的模型

## 一句话提醒

这一周最重要的，不是把每个模块都做到最强，而是把增强版 RAG 主链路补齐成一个完整、稳定、可讲清楚的工程项目。
