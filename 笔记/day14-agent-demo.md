## 先建立今天的最小链路

你今天先把完整过程理解成这条链路：

1. 前端发起聊天请求
2. 后端接收 `messages`
3. 模型先判断要不要调用工具
4. 如果要调用工具，后端执行工具函数
5. 工具结果作为一条 `tool` 消息回传给模型
6. 模型基于工具结果继续生成最终答案
7. 后端把最终答案用流式方式推给前端
8. 前端一边接收，一边实时显示

一句话版：

“前端负责发请求和展示，后端负责调模型和执行工具，模型负责决定何时用工具，最终结果用流式返回。”

## 最小后端心智模型

你今天最适合把服务端逻辑理解成下面这段伪代码：

```js
router.post('/api/chat/agent/stream', async (req, res) => {
  setSSEHeaders(res);

  const conversation = [...req.body.messages];

  res.write(`data: ${JSON.stringify({ type: 'start' })}\n\n`);

  const firstRound = await callModel(conversation, tools);
  const toolCalls = firstRound.tool_calls || [];

  if (toolCalls.length > 0) {
    for (const toolCall of toolCalls) {
      res.write(`data: ${JSON.stringify({ type: 'tool_call', name: toolCall.function.name })}\n\n`);

      const toolMessage = await executeToolCall(toolCall);

      res.write(`data: ${JSON.stringify({ type: 'tool_result', content: toolMessage.content })}\n\n`);

      conversation.push({
        role: 'assistant',
        content: firstRound.content ?? null,
        tool_calls: toolCalls,
      });
      conversation.push(toolMessage);
    }
  }

  const finalStream = await callModelInStreamMode(conversation);

  for await (const delta of finalStream) {
    res.write(`data: ${JSON.stringify({ type: 'assistant_delta', content: delta })}\n\n`);
  }

  res.write('data: [DONE]\\n\\n');
  res.end();
});
```

你今天不一定要和这段伪代码 100% 一样，但脑子里最好有这个图景：

- 先判断是否调工具
- 再执行工具
- 再继续回答
- 最后把回答流式输出

## 今天最值得修的常见坑

### 坑 1：聊天接口和流式接口的请求格式不统一

如果一个接口收：

```json
{ "messages": [...] }
```

另一个接口收：

```json
{ "message": "你好" }
```

那前端和联调逻辑会很容易混乱。

Day 14 最好统一成同一套请求体。

### 坑 2：tool calling 和 streaming 还是两条完全分离的线

如果 `/chat` 负责工具调用，而 `/api/chat/stream` 只负责 Ollama 分片输出，

那你实际上还没有真正做成“Agent 雏形”，只是分别做了两件事。

今天最关键的，就是让它们进入同一条链路。

### 坑 3：SSE 格式不严格

SSE 最稳妥的格式是：

```txt
data: {...}

```

注意点：

- `data:` 后面最好有空格
- 每个事件后面最好是两个换行
- 结束时保持统一 `[DONE]`

### 坑 4：异常时返回格式不一致

正常时是 SSE，报错时却突然 `res.json(...)`，前端就很容易解析崩掉。

更稳妥的方式是：

- 流式接口在异常时也返回 SSE 事件
- 最后依然给一个结束标记或主动结束连接

### 坑 5：前端只会显示正文，不显示过程

如果前端只能显示最后拼出来的文本，

那别人其实看不见工具调用过程。

今天想录一个更像 Agent 的 demo，最好把过程事件也显示出来。

### 坑 6：只在成功路径测试，没测失败路径

你今天至少应该测 3 类情况：

- 正常回答，不触发工具
- 触发工具并成功返回
- 工具调用失败或上游异常
