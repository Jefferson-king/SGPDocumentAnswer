# Day 35｜2026-04-18｜4h

## 今日主题

- 增加问答控制层
- 为不同问题类型设计模板
- 让问答不再只靠一个万能 prompt

验收：

系统已经能区分至少 4 类问题，并按类型走不同模板。

## 适用范围

- 当前项目技术栈：`Node.js + Express + Vue`
- 当前问答状态：已有统一问答 prompt 和基本多轮逻辑
- 当前目标：把问答链路从“统一 prompt 直接问”升级为“带控制层的问答编排”

## 今天做完后，你应该达到的结果

- 问答更像一个有策略的系统，而不是裸调模型
- 不同问题类型的回答风格更稳定
- 你的项目里能明显体现出“控制层设计”能力

## 先用一句话理解

今天要做的，是让系统先理解“这是什么类型的问题”，再决定“应该怎么回答”。

## 为什么今天一定要加控制层

如果只有一个万能 prompt，最容易发生的事就是：

- 总结题答得像事实题
- 对比题答得不够结构化
- 追问题没有承接感
- 每类问题都能答，但都不够稳

一句话版：

“今天加控制层，不是增加复杂度，而是让不同问题走更合适的回答策略。”

## 推荐学习和实战顺序

### 模块 1：先定义问题类型（1h）

今天最适合先把最常见的 4 类问题固定下来：

- `fact`
- `summary`
- `compare`
- `followup`

你今天不一定要做得非常全，但这 4 类已经足够支撑简历表达和产品演示。

推荐理解方式：

- `fact`：问具体事实、是否提到、在哪一页
- `summary`：总结、概括、提炼核心结论
- `compare`：比较区别、优缺点、不同对象差异
- `followup`：承接上一轮上下文继续追问

### 模块 2：做第一版问题分类逻辑（1h）

今天最稳的分类方式，通常不是先上模型分类，而是规则分类。

你可以先从这些特征下手：

- 包含“总结、概括、归纳” -> `summary`
- 包含“区别、差异、对比” -> `compare`
- 包含“刚才、上面、第二点、这部分、它” -> `followup`
- 其余默认 -> `fact`

如果今天时间够，可以再让历史对话参与 followup 判断。

重点是：

- 先把问题分类入口做出来
- 不要今天就在分类精度上过度追求完美

### 模块 3：为每类问题写模板（1.5h）

今天真正体现控制层价值的是模板层。

每一类问题都应该至少控制这些内容：

- 回答结构
- 是否优先给结论
- 是否强调引用
- 是否优先列点

例如：

- `fact`：先结论，再出处
- `summary`：先摘要，再分点总结
- `compare`：按对象或维度对照输出
- `followup`：先承接上一轮，再展开说明

也就是说，今天的目标不是多写几个 prompt，而是让每类问题都有更稳定的回答格式。

### 模块 4：做回归测试（30min）

今天别忘了用 Day 30 的固定测试集回归。

至少确认：

- 每一类题都能命中对应模板
- 模板没有明显跑偏
- followup 不会误判太多

## 今天最值得注意的常见坑

### 坑 1：分类逻辑太复杂

第一版规则分类已经够用。

### 坑 2：只是名字不同，模板内容几乎一样

那控制层就没有真实价值。

### 坑 3：followup 只看当前问题，不看历史

这样多轮场景下效果会不稳。

### 坑 4：今天只做分类，不改回答结构

那用户侧基本感知不到提升。

## 今天的验收标准

- 已经定义至少 4 类问题
- 已经有第一版分类逻辑
- 每类问题已具备不同模板
- 已经做了一轮固定测试集回归
