API /api/qa/ask 返回 500 错误
原因：Ollama embedding 模型名称错误。代码中使用 'all-minilm-l6-v2-f32'，但实际模型名称是 'chroma/all-minilm-l6-v2-f32:latest'。

2. Chroma 启动失败
原因：chroma run --path .\chroma-data 返回 Exit Code: 1，可能由于 Chroma 未正确安装或路径问题（输出有 OpenTelemetry 警告，但不影响运行）。
解决：后来在另一个终端成功启动，运行在 http://localhost:8000。
3. Ollama 服务启动失败
原因：ollama serve 返回 Exit Code: 1，端口 11434 被占用（已有 Ollama 实例运行）。
解决：无需额外操作，已有实例运行在 http://localhost:11434。
4. 后端服务器启动失败
原因：npm run dev 返回 Exit Code: 1，端口 3000 被占用（输出 "Error: listen EADDRINUSE: address already in use :::3000"）。
解决：使用 taskkill /PID <PID> /F 杀掉占用进程（PID 34396 和 27164），然后重启成功，运行在 http://localhost:3000。
5. 前端开发服务器启动失败
原因：npm run dev 返回 Exit Code: 1，端口 5173 被占用。
解决：Vite 自动切换到 5174 端口，成功运行在 http://localhost:5174。
6. 文档上传失败（早期）
原因：documents.js 路由返回 "接口开发中"，未实现上传逻辑；PowerShell 不支持 multipart/form-data。
解决：实现 documents.js 上传逻辑，使用 Node.js 脚本上传成功。
7. favicon.ico 404 错误
原因：前端项目根目录没有 favicon.ico 文件。
解决：不影响功能，无需修复。
8. Vue Router 警告：No match found for location with path "/documents"

解决：不影响问答功能，无需修复（如果需要，可添加路由）。
9. PowerShell Invoke-WebRequest 安全警告
原因：PowerShell 默认阻止执行网页脚本。
解决：添加 -UseBasicParsing 参数或手动确认 "Y"。

## 切片问题
1. 切片问题：如果text是短文本，在进行切片的时候，start =end-overlap到最后一段不再前进，导致死循环，最终把Node内容撑爆，导致前端报错。解决方法：增加判定：如果end>text.length，则直接退出
2. 模型的输入上线问题：切的chunk过长以及数据清洗不彻底进行embedding时超过了模型的处理输入上限。解决方法：数据清洗去掉页码，连续空白，tab等以及减小chunk大小。
