## 什么叫“进入成品期”

前面的 Day 14 到 Day 21 更像是在逐步验证能力：

- 聊天能不能跑
- tool calling 能不能跑
- SSE 能不能跑
- 文档能不能入库
- RAG 能不能回答

但从 Day 22 开始，你要解决的问题不再只是“能不能跑”，而是：

- 用户从哪里上传文件
- 用户从哪里提问
- 用户从哪里看文档
- 接口和页面如何长期维护
- 后面做引用、多轮、评测时放在哪里最合理

一句话版：

“Day 22 的核心，不是增加单点能力，而是给后面的一整周产品化开发搭骨架。”

## 今天为什么一定要先做结构

如果你今天跳过这一步，后面很容易出现这些问题：

- 上传页写完了，结果不知道文档列表页复用哪些状态
- 问答页写完了，后面加引用展示时组件层级很乱
- 后端接口一会儿挂在 `/api/chat`，一会儿挂在 `/api/retrieval`，一会儿又单独写脚本
- 文件上传、入库状态、文档列表三块逻辑互相缠绕

真正的正式项目，最先要稳定的不是 UI，而是信息架构和工程架构。

## 先建立今天的最小产品蓝图

你今天最适合把正式产品理解成 3 个页面 + 4 条主接口：

### 页面

- 上传页：负责上传 PDF/TXT，并显示处理状态
- 问答页：负责发起问题、展示流式回答、展示引用
- 文档列表页：负责查看已上传文档、状态、基础信息

### 接口

- `POST /api/documents/upload`
- `GET /api/documents/:id/status`
- `GET /api/documents`
- `POST /api/qa/ask`

一句话版：

“先把页面和接口的责任边界定清楚，后面每天只是在这个骨架上继续补肉。”

## 推荐学习和实战顺序

### 模块 1：先确定产品主流程（40min）

你今天先别画目录，先回答用户主流程：

1. 用户先进入哪个页面
2. 上传成功后会看到什么
3. 文档处理完成后如何进入问答
4. 问答结果如何回到文档来源

你应该先把产品主链路定成这样：

上传文档 -> 查看处理状态 -> 文档入库完成 -> 进入问答页提问 -> 查看回答与引用 -> 回到文档列表确认来源

这一阶段的目标不是写代码，而是回答一句话：

“这个产品从用户视角到底怎么走完一圈。”

### 模块 2：确定页面信息架构（1h）

当主流程清楚后，再定页面。

上传页至少要有：

- 拖拽/选择文件区域
- 文件校验提示
- 上传进度或处理中状态
- 成功/失败反馈

问答页至少要有：

- 输入框
- 消息列表
- 流式回答区
- 引用区

文档列表页至少要有：

- 文件名
- 上传时间
- 状态
- 文档类型

这一阶段最关键的是：

- 页面先清楚职责，不要急着美化
- 同一个状态不要散落在多个页面重复维护

### 模块 3：确定接口边界（1.5h）

今天一定要把接口先定义清楚，再写页面。

最推荐你今天先把 4 条接口定成这样的责任：

- 上传接口：只负责接收文件、创建任务、返回文档 ID
- 入库状态接口：只负责告诉前端当前处理到哪一步
- 文档列表接口：只负责返回已存在的文档列表
- 问答接口：只负责基于知识库回答问题

今天最值得你先写出来的不是实现，而是接口契约，例如：

```json
POST /api/documents/upload
{
  "ok": true,
  "data": {
    "documentId": "doc_001",
    "status": "queued"
  }
}
```

```json
GET /api/documents
{
  "ok": true,
  "data": [
    {
      "id": "doc_001",
      "fileName": "demo.pdf",
      "status": "ready"
    }
  ]
}
```

### 模块 4：设计目录结构（1h）

这一步你最适合写出一版正式目录，而不是继续把所有逻辑塞在旧文件里。

推荐最小目录思路：

src/

你今天不一定非要把所有文件都建出来，但至少要让后面每块功能都有固定位置。

## 今天最值得注意的常见坑

### 坑 1：一边写页面一边想结构

这样最容易导致目录和组件不断返工。

### 坑 2：接口职责不清

上传接口如果同时负责“上传 + 入库 + 返回完整文档列表”，后面会越来越难维护。

### 坑 3：页面太多

Day 22 先控制在 3 个核心页面，不要一口气再拆设置页、统计页、调试页。

### 坑 4：项目里同时存在“脚本流”和“产品流”

实验脚本可以保留，但正式页面和正式接口必须有独立主链路。
